## Overview

The `plantilla.html` file uses a simple variable replacement system. The Python script searches for placeholders wrapped in double curly braces and replaces them with generated content.

<Info>
All replacements happen in memory using Python's built-in `str.replace()` method.
</Info>

## Available Variables

### `{{TITULO}}`

<ParamField path="{{TITULO}}" type="string">
  The story title generated by the AI.
  
  **Location in template**: Line 117 (inside `<h1>` tag)
  
  **Source**: Extracted from AI response using regex pattern `<TITULO>(.*?)</TITULO>`
  
  **Fallback**: `"Historia sin título"` if extraction fails
</ParamField>

**Example output**:
```html
<h1>El Guardián del Faro de las Sombras</h1>
```

**Usage in template**:
```html
<h1>{{TITULO}}</h1>
```

---

### `{{HISTORIA}}`

<ParamField path="{{HISTORIA}}" type="string">
  The main story text (2-3 paragraphs) generated by the AI.
  
  **Location in template**: Line 120 (inside `<p>` tag within `.historia-texto` div)
  
  **Source**: Extracted from AI response using regex pattern `<HISTORIA>(.*?)</HISTORIA>`
  
  **Fallback**: `"Error al generar la historia."` if extraction fails
</ParamField>

**Example output**:
```html
<p>En el confín del universo conocido, donde las estrellas mueren en silencio...</p>
```

**Usage in template**:
```html
<div class="historia-texto">
    <p>{{HISTORIA}}</p>
</div>
```

<Note>
The story content may include line breaks and special characters. The template preserves all formatting.
</Note>

---

### `{{IMAGEN_URL}}`

<ParamField path="{{IMAGEN_URL}}" type="string" format="url">
  A Picsum.photos URL with a unique UUID seed for reproducible random images.
  
  **Location in template**: Line 118 (`src` attribute of `<img>` tag)
  
  **Format**: `https://picsum.photos/seed/{uuid}/600/350`
  
  **Dimensions**: 600×350 pixels
</ParamField>

**Example output**:
```html
<img src="https://picsum.photos/seed/a1b2c3d4e5f6/600/350" alt="Imagen del día" class="imagen-dia">
```

**Usage in template**:
```html
<img src="{{IMAGEN_URL}}" alt="Imagen del día" class="imagen-dia">
```

<Tip>
The UUID changes with every script run, ensuring different images daily while keeping them stable (same seed always returns the same image).
</Tip>

---

### `{{MENU}}`

<ParamField path="{{MENU}}" type="string" format="html">
  Dynamically generated HTML for the sidebar navigation menu.
  
  **Location in template**: Line 113 (inside `<nav id="sidebar">`)
  
  **Source**: Built from `historial.json` data structure
  
  **Structure**: Month headers with lists of story links
</ParamField>

**Example output**:
```html
<h3 class="mes-titulo">Marzo 2026</h3>
<ul class="lista-historias">
    <li><a href="historia-2026-03-04.html">El Guardián del Faro de las Sombras</a></li>
    <li><a href="historia-2026-03-03.html">El Guardián del Laberinto Cuántico</a></li>
    <li><a href="historia-2026-03-02.html">El Código de las Estrellas</a></li>
</ul>
<h3 class="mes-titulo">Febrero 2026</h3>
<ul class="lista-historias">
    <li><a href="historia-2026-02-28.html">Error</a></li>
    <li><a href="historia-2026-02-26.html">El Eco de los Días Perdidos</a></li>
</ul>
```

**Usage in template**:
```html
<nav id="sidebar">
    <button id="close-btn" onclick="toggleMenu()">✖</button>
    <h2>Historial</h2>
    {{MENU}}
</nav>
```

<Warning>
The menu HTML must be valid and properly escaped. Titles containing special characters (quotes, angle brackets) could break the HTML structure.
</Warning>

## Replacement Process

The script replaces variables in a specific order:

<Steps>
  <Step title="Read template file">
    ```python
    with open('plantilla.html', 'r', encoding='utf-8') as archivo:
        contenido_html = archivo.read()
    ```
  </Step>
  
  <Step title="Replace title">
    ```python
    contenido_html = contenido_html.replace('{{TITULO}}', nuevo_titulo)
    ```
  </Step>
  
  <Step title="Replace story">
    ```python
    contenido_html = contenido_html.replace('{{HISTORIA}}', nueva_historia)
    ```
  </Step>
  
  <Step title="Replace image URL">
    ```python
    contenido_html = contenido_html.replace('{{IMAGEN_URL}}', url_imagen)
    ```
  </Step>
  
  <Step title="Replace menu">
    ```python
    contenido_html = contenido_html.replace('{{MENU}}', menu_html)
    ```
  </Step>
</Steps>

<Note>
The replacement order doesn't matter in this case since no variable contains another variable.
</Note>

## HTML Structure

The template includes several key structural elements:

### Page Layout

```html
<body>
    <button id="menu-btn" onclick="toggleMenu()">☰</button>
    
    <nav id="sidebar">
        <!-- Menu content here -->
    </nav>
    
    <div class="container">
        <!-- Story content here -->
    </div>
    
    <script>
        function toggleMenu() {
            const sidebar = document.getElementById('sidebar');
            sidebar.classList.toggle('abierto');
        }
    </script>
</body>
```

### CSS Classes Reference

<AccordionGroup>
  <Accordion title="Menu Classes">
    - `.mes-titulo`: Month header styling (color, border)
    - `.lista-historias`: Story list container
    - `.lista-historias li`: Individual story items
    - `.lista-historias a`: Story links with hover effects
    - `#sidebar`: Fixed sidebar container
    - `#sidebar.abierto`: Sidebar visible state
    - `#menu-btn`: Hamburger menu button
    - `#close-btn`: Close button inside sidebar
  </Accordion>
  
  <Accordion title="Content Classes">
    - `.container`: Main content card with shadow and rounded corners
    - `.imagen-dia`: Story image styling
    - `.historia-texto`: Story text container with paragraph styling
    - `.footer`: Small footer text
  </Accordion>
</AccordionGroup>

### JavaScript Functions

<ParamField path="toggleMenu()" type="function">
  Toggles the sidebar visibility by adding/removing the `abierto` class.
  
  **Called by**: Menu button (`#menu-btn`) and close button (`#close-btn`)
  
  **Implementation**:
  ```javascript
  function toggleMenu() {
      const sidebar = document.getElementById('sidebar');
      sidebar.classList.toggle('abierto');
  }
  ```
</ParamField>

## Customization Tips

<Tip>
**Adding new variables**: Simply add `{{YOUR_VARIABLE}}` to the template and include a corresponding `.replace()` call in the script:

```python
contenido_html = contenido_html.replace('{{YOUR_VARIABLE}}', your_value)
```
</Tip>

<Warning>
**Special characters**: Make sure to escape HTML entities in dynamic content to prevent breaking the page structure:

```python
import html
safe_title = html.escape(nuevo_titulo)
```
</Warning>
